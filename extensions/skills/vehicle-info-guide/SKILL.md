# 车型信息引导 Skill（T1）

## 职责

引导车主提供车型信息，根据当前项目需要的精度等级（L1/L2/L3）选择合适的引导策略。

## 精度等级定义

| 精度 | 内容 | 适用场景 |
|------|------|----------|
| L1 简单车型 | 品牌+车系（如"宝马X3"） | 洗车、简单美容、通用咨询 |
| L2 精确车型 | 品牌+车系+年款+排量（如"宝马X3 2022 2.0T"），必要时可带 car_model_id | 常规保养、标准配件更换、一般报价 |
| L3 VIN | 完整 17 位车架号 | 精准报价、配件兼容性校验、高风险维修、生命周期项目推荐 |

## 执行步骤

1. 先阅读本 skill 中与当前问题直接相关的 reference 文档：
   - VIN 价值和获取方式：`read <skill-fs-dir>/references/VIN价值说明与获取策略.md`
   - 精度分级与服务映射：`read <skill-fs-dir>/references/无需VIN的服务清单.md`
2. 判断当前项目需要的精度等级（L1/L2/L3）
3. 如果已有车型信息满足需求，直接使用，不再追问
4. 如果需要更高精度，按以下分流选择 tool：
   - 用户给了车型描述（品牌/车系/年款等）→ 调用 `fuzzy_match_car_info`
   - 用户说"我那辆车"或可能已绑定车辆 → 调用 `list_user_cars`
   - 用户没有提供足够车型信息 → 调用 `ask_user_car_info` 引导补充

## Tools

- `fuzzy_match_car_info`：根据车主描述模糊匹配车型，获取 car_model_id
- `list_user_cars`：查看车主已绑定的车辆列表
- `ask_user_car_info`：引导车主提供更详细的车型信息
- `tire_image_recognize`（待注册）：轮胎照片识别规格

## 三种获取策略

1. **展示价值**：说明有 VIN 后的好处（精准报价、生命周期项目清单）
2. **引导上传**：提供上传链接，引导车主自行上传
3. **线下协助**：建议预约简单项目（如洗车），让商户师傅协助录入

## 轮胎项目特殊场景

轮胎相关项目（换胎、补胎、轮胎检测）除车型信息外，还可能需要轮胎规格（如 225/45R17）。
- "是否需要 tire_spec"不由 skill 静态判断，由 match_project / get_project_price 等 API 返回 missing_fields: ["tire_spec"]
- 收到 missing_fields 后，引导车主补充轮胎规格或上传完整轮胎照片自动识别
- 调用 `tire_image_recognize` tool（待注册）进行照片识别

## 降级路径

车主拒绝或无法提供 VIN 时：
- 明确告知精度不足可能带来的风险
- 降级到 L2（品牌+车系+年款+排量）继续服务
- 记录当前精度，后续如需升级可再次引导

## 完成标准

- L1：已知品牌+车系
- L2：已知品牌+车系+年款+排量（可带 car_model_id）
- L3：已获取完整 VIN
- 降级完成：应升 L3 但用户拒绝，已告知风险并记录 L2
