# 需求挖掘方法论

在 Discovery 阶段，需要逐步建立信任并挖掘可落地的养车需求。

## 执行步骤

1. 使用 `read` 工具读取与当前问题相关的 reference 文档，再作答：
   - 需求挖掘技巧 → `read` `<skill-fs-dir>/references/挖掘需求技巧.md`
   - 养车分析框架与技巧 → `read` `<skill-fs-dir>/references/养车秘笈.md`
2. 基于文档内容和当前对话上下文作答。
3. 不要脱离 reference 臆测平台规则、业务边界或方法论细节。

## 三大核心目标

1. **获取车辆信息**：引导车主提供 VIN 码或车型/车龄/里程
2. **识别消费偏好**：了解车主最看重省钱、省心还是省时
3. **挖掘首个可落地项目**：找到车主当下最可能尝试的第一个项目

## 获取 VIN 的三种方式

- 说明有 VIN 后的好处，展示示例车架号对应的生命周期项目清单
- 引导车主按专用链接自行上传 VIN
- 建议车主下次洗车时让门店师傅协助录入

## 需求挖掘的三大来源

1. **直接表达**：车主明确说出需求（"我要换轮胎"）——判断真实性，直接推进
2. **潜在/未知需求**：车主因缺乏专业知识不知道该做什么——根据车型/车龄/里程，结合生命周期保养清单，主动引导高优先级项目
3. **未转化需求**：车主有需求但未打算通过话痨预订——通过展现专业性、对比现有方案的优势，引导"第一次尝试"
