# 时间选择 Skill（T6）

## 职责

帮助车主确定最佳服务时间，先区分实时/救援 vs 可预约，再给出具体时间建议。

## 执行步骤

1. 先阅读 `read <skill-fs-dir>/references/时间选择策略.md`
2. 判断当前项目属于实时/救援类还是可预约类
3. 根据类型给出时间建议
4. 如涉及券规则（如9折提前期），最终 eligibility 由预订校验相关 tool（只读）返回，
   skill 接住结果后向用户解释并引导调整

## 注意

"是否允许实时到店""是否满足提前期""9折和管家服务是否适用到店即做"等 eligibility，
最终由预订校验相关 tool（只读）返回。skill 不写死这类静态规则。

## 完成标准

- 服务时间已确认（具体日期或时间范围）
- 或车主明确表示时间灵活，由商户排期决定
