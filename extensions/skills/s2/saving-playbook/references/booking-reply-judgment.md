# 预订确认回复判断

`confirm_booking` 返回车主的原始回复文本，根据文字内容判断意图：

- **确认预订**（"确认"、"没问题"、"可以"、"好的下单吧"等）→ 告知车主预订已提交，skill 完成
- **想取消**（"不用了"、"算了"、"取消"等）→ 告知车主已取消，skill 完成
- **想换商户**（如"换一家店"、"看看其他商户"）→ 退出当前预订流程，转入 `merchant-selection` 商户选择流程，skill 完成
- **想改时间**（如"改到周末"、"下周再去"）→ 更新 booking_time，重新调用 `confirm_booking`，回到本步骤
- **想改项目**（如"把洗车去掉"、"加上空调清洗"）→ 退出当前预订流程，转入 `project-demand-clarifier` 项目选择流程，skill 完成
- **想改价格**（如"能便宜点吗"、"价格太高了"、"500行不行"）：
  - **standard 模式** → 告知车主价格由商户报价决定，无法在预订时修改。如果觉得价格不合适，可以换商户或换项目；也可以给个目标价，帮你问多家商户（转为 commission）
  - **commission 模式** → 更新 price（车主出的一口价），重新调用 `confirm_booking`，回到本步骤
- **想加备注**（如"帮我备注一下车在地下二层"、"到店前电话联系我"）→ 更新 remark，重新调用 `confirm_booking`，回到本步骤
- **无关内容** → 简要引导车主确认或说明想调整什么，回到本步骤

注意：plan_mode 由 agent 根据当前条件（项目、商户、券型等）自动判断，车主不直接选择或修改模式。当车主调整了商户、时间或项目后，agent 应重新评估 plan_mode 是否需要变更。
注意：standard 模式下 price 来自商户报价，不允许修改；commission 模式下 price 是车主出的一口价，允许调整。
