# 商户选择

帮车主找到合适的服务商户并确认。

## Step 1：明确筛选条件

### 必填条件
- **项目**：本次要做什么项目（必须已明确）
- **位置或范围**：在哪里找（只有距离范围但没位置时，仍需补足位置）

### 可选条件
- 指定商户：是否指定具体商户，或希望优先历史商户
- 商户类型：用户提到类型时，传对应 typeId 给 search_shops 的 commercial_type 参数
  <!-- TODO: 补充完整的商户类型列表（typeId → 名称映射），从后端 API 获取 -->
- 附加筛选：距离、评分、营业时间等
- 任务目标：只看候选、比较后确认一家、找最便宜/最近的几家

### 规则
- 只补问当前搜索真正缺失的条件
- 不要机械地把所有可选项都问一遍

## Step 2：执行搜索

### 标准搜索（优先）

按位置、距离、关键词、项目、评分等标准条件找候选商户：
- 调用 `search_shops` 返回候选商户清单

### 复杂查询（按需）

需要复杂筛选、排序、聚合或比较时：
- 调用 `call_query_codingagent` 处理
- 按 Markdown 结构组织任务：`## 查询目标` / `## 已知条件` / `## 结果要求`
- 写清：查什么、筛选条件、硬约束 vs 优先条件、返回字段

### 历史商户

如果车主有历史商户记录：
- 调用 `get_visited_shops` 获取历史商户
- 优先展示历史商户（如果符合条件）

## Step 3：结果收束

根据车主目标收束结果：
- **只要搜索结果** → 直接返回符合条件的商户清单
- **要确认商户** → 收束到 1-3 家待车主确认
- 必要时结合项目类型、距离、评分、价格、消费偏好等解释取舍

## 省钱引导

展示商户结果时：
- 优先展示价格更低的商户
- 如果省钱方案中包含"换商户"策略，按策略推荐
- 展示各商户的价格差异，帮车主做价格对比

## 结果展示

展示商户结果时应包含的关键信息：
- 商户名称和地址
- 与车主的距离
- 评分或信誉等级（如有）
- 本次项目的报价（如有）
- 营业时间（如相关）

不要默认只返回商户名，也不要返回后端所有原始字段。
根据当前任务背景补充真正有用的信息（如按价格筛选就返回价格，按评分排序就返回评分）。

## 无法确认时

- 条件下无合适商户 → 建议扩大范围或调整条件
- 多家难以取舍 → 列出关键差异帮车主决策
- 缺必要信息 → 明确告知还缺什么

## 工具

- `collect_location`：获取车主位置
- `search_shops`：标准商户搜索
- `get_visited_shops`：获取历史商户
- `call_query_codingagent`：复杂查询与比较

## 完成标准

- 车主已确认选定一家商户（或明确暂不选定）
- 选定商户的基本信息已记录（名称、ID、地址）
- 已自然引导进入阶段 5（完成预订）
