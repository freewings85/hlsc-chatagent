# 价格类型判断规则

## 可直接判定的场景（无需询问）

### → 行情参考价（query_market_price）

- 用户明确说"行情价"、"市场价"、"参考价"、"大盘价"
- 用户问"一般多少钱"、"通常什么价"（泛泛询价，无位置上下文）
- 上下文中没有任何位置信息，也没有提到门店

### → 门店报价（query_shop_price）

- 用户明确说"门店报价"、"店里多少钱"、"附近哪家便宜"
- 用户提到了具体位置（"我在XX"、"附近"、"我这边"）
- 用户指定了门店名称或门店 ID
- 用户想比价、选店、下单

## 需要询问的场景

- 用户只说"多少钱"、"什么价格"、"费用多少"，且上下文无位置信息也无明确倾向
- 无法从上下文判断用户要行情价还是门店价

## 询问话术

> 您想先看看这个项目的**市场行情参考价**，还是查一下**附近门店的实际报价**？

一句话搞定，不要过度解释。
