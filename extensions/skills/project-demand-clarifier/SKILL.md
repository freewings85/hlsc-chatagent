# 项目需求澄清 Skill（T3 前半）

## 职责

项目确认的总控器。负责：
- 判断当前属于哪种项目确认场景
- 按场景加载对应方法论
- 调用 tool 完成项目匹配
- 将结果收敛为后续阶段可消费的"项目确认结果"

不负责：车型精度裁决、报价、商户推荐、方案制定、复杂诊断。

## 场景索引

| 场景 | 典型表现 | 对应 reference |
|------|----------|----------------|
| 直接表达 | 车主明确说出项目或试探询价 | `直接表达需求.md` |
| 模糊意图 | 故障现象描述、效果期望、关键词触发 | `模糊意图澄清.md` |
| 潜在需求 | 车主未提需求但车况可能需要关注 | `潜在需求引导.md` |
| 事故/现场检查 | 碰撞事故分流、无法远程确认的项目 | `事故与现场检查处理.md` |
| 管家项目提取 | 车主有管家服务，需提取本次项目 | `管家项目提取.md` |

## 执行步骤

1. **判断场景**：根据车主当前表达，判断属于上述哪种场景
2. **加载方法论**：读取对应 reference
   - `read <skill-fs-dir>/references/直接表达需求.md`
   - `read <skill-fs-dir>/references/模糊意图澄清.md`
   - `read <skill-fs-dir>/references/潜在需求引导.md`
   - `read <skill-fs-dir>/references/事故与现场检查处理.md`
   - `read <skill-fs-dir>/references/管家项目提取.md`
3. **调用 tool 确认项目**：通过项目匹配 tool 将车主描述映射到标准项目
4. **收尾**：项目基本确认后，读取 `read <skill-fs-dir>/references/附加要求确认.md`，收集附加要求
5. **判断跳转**：
   - tool 返回 missing_fields 包含车型相关字段 → 回溯车型信息引导
   - tool 返回结果表明需要现场确认 → 进入商户选择做现场检查
   - 需要故障诊断或项目推荐 → 委托项目推荐 subagent
   - 需要知识库辅助意图澄清 → 调用知识库搜索 tool

## 车型精度不足时的处理

本 skill 不负责判断车型精度等级，不直接调用车型工具。
当 tool 返回 `missing_fields` 包含车型相关字段时，直接补齐当前步骤所需的车型信息。

## Tools

- `match_project`：将车主描述匹配到标准项目
- `knowledge_base_search`（待注册）：知识库搜索辅助意图澄清

## Subagents

- `call_recommend_project`：故障诊断、项目推荐（根据车型/车龄/里程推荐应做项目）

## 完成标准

- 至少一个具体项目已确认（项目名称 + 基本规格）
- 如有多个候选项目，已与车主确认优先级
- 附加要求已记录（如有）
- 如项目无法远程确认，已明确转商户选择做现场检查
- 如车型精度不足，已标记需回溯车型信息引导
