# 知识库澄清与模糊意图

## 模糊意图的澄清

车主描述模糊时，不要只靠对话追问，可结合知识库搜索和车辆生命周期项目清单来辅助澄清。

场景示例：
- 故障现象描述："车有点响""感觉刹车不太好""车开起来有点抖"
- 想要实现某种效果："车子脏了想搞干净""空气也好一些"
- 关键词触发："朋友的车搞了木地板，漂亮，也想搞"
- 服务边界需要确认："车子搞个大保养，底盘也整一下"

澄清方法：
1. 调用知识库搜索相关 tool 搜索与车主描述相关的知识
2. 结合车辆生命周期项目清单缩小可能的项目范围
3. 根据搜索结果向车主确认具体项目或项目类型
4. 引导车主提供更多细节（什么时候、哪个位置、什么情况下）
5. 如仍无法确定，委托项目推荐 subagent 基于症状推荐
6. 如完全无法远程判断，建议到店检查

注意：
- 知识库搜索是辅助手段，结果用于帮助理解车主需求，不是直接作为回答
- 对车主用不专业的术语，先理解意图再映射到标准项目
