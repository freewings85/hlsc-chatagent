# 直接表达需求

车主明确说出项目需求时的处理方法。

## 场景示例

- "我要换刹车片"
- "帮我约个保养"
- "刹车片换换多少钱？"（试探性询价也算直接表达）

## 确认要点

1. 确认具体项目名称和规格
2. 确认是否有配件品牌/档次偏好
3. 确认数量（如"前刹车片"还是"前后都换"）

## 流程跳转

- 若项目匹配相关 tool 返回 missing_fields 包含车型相关字段（车型精度不足）→ 回溯车型信息引导
- 如偏好尚未确认 → 确认偏好
- 确认后 → 进入省钱目标确认

注意：
- 本 skill 不直接调用车型工具，车型升级统一由车型信息引导 skill 负责
- 试探性询价（"多少钱"）不等于已决定做，需要判断真实意图
- 车主可能用不专业的术语描述项目，需要通过项目匹配 tool 映射到标准项目名称
