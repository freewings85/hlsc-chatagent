# 项目确认主流程

## 直接表达需求（T3.1）

车主明确说出项目需求时的处理方法。

场景示例：
- "我要换刹车片"
- "帮我约个保养"
- "刹车片换换多少钱？"（试探性询价也算直接表达）

确认要点：
1. 确认具体项目名称和规格
2. 确认是否有配件品牌/档次偏好
3. 确认数量（如"前刹车片"还是"前后都换"）
4. 确认是否有附加要求（当天完成、指定商户等）

流程跳转：
- 若项目匹配相关 tool 返回 missing_fields 包含车型相关字段（车型精度不足）→ 回溯车型信息引导
- 如偏好尚未确认 → 确认偏好
- 确认后 → 进入省钱目标确认 → 省钱方法选择
- T3 不直接调用车型工具，车型升级统一由车型信息引导 skill 负责

注意：
- 试探性询价（"多少钱"）不等于已决定做，需要判断真实意图
- 车主可能用不专业的术语描述项目，需要通过项目匹配 tool 映射到标准项目名称

## 引导潜在需求（T3.2）

车主因缺乏专业知识而不清楚自己需要什么时：
1. 根据车型、车龄、里程，结合车辆生命周期项目清单
2. 主动沟通优先级高、高频的、刚性的项目
3. 委托项目推荐 subagent 获取推荐
4. 引导保养维修的管家服务

## 附加要求确认

项目基本确认后，判断是否有不相关或完全不可能的额外需求，否则记录下来作为项目预订需求的一部分。
比如：时间要求、换胎是否送动平衡和四轮定位等。

不需要一次问完，在后续推进中自然收集即可。
