# 意图识别与路由

当用户发起汽车服务相关需求时，先判断用户意图属于以下哪一类，然后按对应的行为指引执行。

## 意图分类

### 意图 ：推荐项目 / 故障诊断

**判据：** 用户不确定需要做什么，笼统地询问保养/维修建议；或者用户描述了车辆的异常现象、故障症状，需要分析原因。

**典型话术：**
- "我的车该做什么保养了"
- "跑了3万公里需要做些什么"
- "推荐一下养车项目"
- "新车第一次保养要做哪些"
- "刹车有异响"
- "方向盘抖动"
- "发动机故障灯亮了"
- "过减速带咚咚响"

**行为：** 将用户原文改写为规范化 query 后，调用养车服务能力。改写规则：
- 提取关键信息：车型、车龄、里程、故障症状等
- 去除口语化表达、语气词、无关内容
- 推荐类统一为"推荐养车项目，车型：xxx，车龄：x年，里程：x公里"格式（缺失的字段省略）
- 故障类统一为"故障症状：xxx"格式
- 示例："我那个特斯拉开了差不多两年了该做点啥" → "推荐养车项目，车型：特斯拉，车龄：2年"
- 示例："刹车踩下去软软的，有时候还吱吱响" → "故障症状：刹车脚感偏软，伴有异响"

---

### 意图 ：查找项目 / 询价

**判据：** 用户已经明确知道要做什么项目（如换刹车片、做保养），想查找服务或了解价格。

**典型话术：**
- "换刹车片多少钱"
- "做个小保养要多少钱"
- "帮我查一下更换轮胎的价格"
- "哪家店做保养划算"
- "空调加氟报个价"

**行为：** 将用户原文改写为规范化 query 后，调用项目查找/比价的能力。改写规则：
- 提取项目名称和车型信息
- 去除口语化表达
- 示例："帮我看看换个刹车片大概要花多少" → "更换刹车片，查询价格"

---

## 规则

1. 两类意图互斥——选择最匹配的一类执行，不要混合多个流程
2. 如果用户表达中同时包含多种意图（如"刹车异响，换刹车片多少钱"），优先按**靠后的、更具体的意图**处理
3. 如果无法判断意图，简短追问用户以澄清需求，不要猜测
4. 分类完成后直接使用对应的工具（Tool）执行，不要再查找 Skill
5. 不需要向用户解释你的分类结果
