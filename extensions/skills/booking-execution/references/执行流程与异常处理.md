# 执行流程与异常处理

## 预订确认流程

### 1. 汇总预订参数

从前置任务中收集以下信息：
- plan_mode：预订模式（过渡型/标准/竞标/保险/管家/打包）
- project_ids：项目 ID 列表（来自 match_project）
- shop_ids：商户 ID 列表（来自 search_nearby_shops 或 get_visited_shops）
- car_model_id：车型 ID（来自上下文）
- price：预订价格（来自报价查询结果）
- booking_time：到店时间（来自时间确认步骤）
- remark：备注（默认为空，车主可在确认时补充）

### 2. 发送给前端确认

调用 `create_booking_order`，将预订参数通过 interrupt 发给前端渲染确认卡片。

### 3. 处理用户回复

- **confirmed** → 前端负责创建真实订单，告知车主预订已提交
- **cancelled** → 告知车主已取消，询问是否需要调整方案
- **adjust** → 根据用户回复调整参数，重新调用 `create_booking_order`

## 异常处理

- 确认流程中断（网络、超时等）→ 告知车主异常，建议重试
- 用户回复无法理解 → 引导车主明确是否确认，还是想调整哪项信息
