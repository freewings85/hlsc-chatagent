# 执行流程与异常处理

## 按 plan_mode 分流执行

### T7-A 过渡型
- 生成简化执行请求
- 过渡型方案通常不涉及完整的购券-下单流程
- 可能只是"生成联系方式""推送给商户""预订简单项目"

### T7-B 标准预订
- 确认购券已完成
- 提交标准预订订单
- 推送给对应商户
- 等待商户确认

### T7-C 一口价竞标
- 确认购券和竞标订单已生成
- 推送到符合条件的商户
- 监控抢单状态
- 抢单成功后推送完整预订信息
- 超时处理

### T7-D 保险特殊竞标
（业务侧待补充执行流程）

### T7-E 管家服务
（业务侧待补充执行流程）

### T7-F 打包拆解
- 按子项目顺序分别执行
- 每个子项目走各自 plan_mode 对应的执行路径
- 需 execution-planner-agent 支持复杂编排

## 异常处理

- 购券失败 → 告知原因，建议调整券型或时间
- 订单生成失败 → 检查缺失字段（由 tool 返回 missing_fields），引导补充
- 商户无响应 → 建议更换商户或调整时间
- 竞标超时 → 调用 handle_bidding_timeout，与车主商议调整
- 执行请求不完整 → 由 submit_execution_request 返回 missing_fields，引导补充
