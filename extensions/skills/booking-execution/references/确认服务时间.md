# 确认服务时间

在调用 `create_booking_order` 之前，必须先与车主确认服务时间。无论车主是否有明确时间意向，都需完成以下判断：

## 1. 判断服务紧急程度

- **实时服务**（马上到店）：车主需要立即服务，如救援类项目（搭电、拖车、换胎等）。救援类项目始终视为实时需求，必须将省时作为最高优先级。部分救援类项目即便商户已休息，也需要有办法触达商户并及时反馈车主。
- **可预订服务**（可等待几天）：非紧急项目，车主可选择合适的日期预订。

## 2. 可预订服务的时间建议

对于可预订的服务，主动提醒车主以下注意事项：
- 避开旺季高峰期和节假日前几天，排队等待时间可能较长
- 避开商户休息时间，确保到店时商户正常营业
- "话痨AI预订9折"优惠需提前至少 2 天预订才可使用

## 3. 可预订服务的执行注意事项

确定预订时间后，执行预订方案时还需注意：
- 如果需要在线联系商户，需考虑商户作息时间，非营业时段可能联系不上
- 旺季期间商户在线回复可能较慢，需提前告知车主

## 4. "话痨AI预订9折"限制

"话痨AI预订9折"优惠不适用于实时到店马上服务的场景——大部分项目需要商户提前准备零部件，无法即时完成。

## 完成标准

满足以下任一条件即可继续执行：
- 到店时间已确认（具体日期或时间范围）
- 或车主明确表示时间灵活，由商户排期决定

## 注意

"是否允许实时到店""是否满足提前期""9折和管家服务是否适用到店即做"等 eligibility，
最终由预订校验相关 tool（只读）返回。skill 不写死这类静态规则。

紧急/救援类场景中，默认优先时效和可执行性；此时不需要单独为了消费偏好开启一个额外流程。
