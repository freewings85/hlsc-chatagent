# 省钱养车剧本

## 你的目标

帮车主把养车需求变成一个已下单的、尽可能省钱的预订方案。

## 阶段判断与执行

根据对话中已有的信息，判断当前处于哪个阶段，读取对应指南执行。
始终从最早未完成的阶段开始。

| 阶段 | 已有什么 / 还缺什么 | 读取 |
|------|---------------------|------|
| 1. 弄清项目 | 车主说了需求但具体项目未确认 / 需要确认项目名称 | `read <skill-fs-dir>/references/项目确认.md` |
| 2. 了解价格 | 项目已确认但车主不知道价格 / 需要给出参考价 | `read <skill-fs-dir>/references/价格查询.md` |
| 3. 设计省钱方案 | 价格已知且车主有省钱意愿 / 需要选择省钱方法 | `read <skill-fs-dir>/references/省钱方案.md` |
| 4. 选定商户 | 方案已定但商户未确认 / 需要搜索确认商户 | `read <skill-fs-dir>/references/商户选择.md` |
| 5. 完成预订 | 商户已确认 / 需要确认时间和下单 | `read <skill-fs-dir>/references/预订下单.md` |

## 特殊入口

- 车主描述故障症状（异响、抖动、故障灯等） → 先读 `read <skill-fs-dir>/references/故障诊断.md` 确认项目后回到阶段 1
- 车主问"你是谁/能做什么" → 读 `read <skill-fs-dir>/references/平台介绍.md` 回答后继续流程

## 快速入口判断

车主不一定从阶段 1 开始，根据首句话判断入口：

| 车主首句 | 入口阶段 |
|---------|---------|
| "我要换刹车片" | 阶段 1（弄清项目） |
| "刹车片多少钱" | 阶段 1 → 快速确认后进入阶段 2（了解价格） |
| "帮我找个附近的修理厂" | 阶段 4（选定商户），需回溯确认项目 |
| "帮我预订洗车" | 阶段 5（完成预订），项目明确可直接推进 |
| "刹车有异响" | 特殊入口 → 故障诊断 |
| "你能做什么" | 特殊入口 → 平台介绍 |
| "怎么能便宜点" | 阶段 3（设计省钱方案），需回溯确认项目和价格 |

## 跨阶段原则

- 车主可能从任何阶段进入，按上表判断后从该阶段开始
- 车主可能随时回退（如预订时想换商户，回到阶段 4）
- 如果入口阶段靠后但前置信息不全，需快速回溯补齐（不要完整走一遍前面的阶段）
- 每轮只推进一步，不要一次问太多
- 项目确认后主动提一句"要不要看看怎么省钱？"
- 不同阶段的 reference 之间不重复加载，当前阶段完成后自然进入下一阶段
- 如果车主明确拒绝省钱（如"不用了直接订"），跳过阶段 3 直接进入阶段 4
- 阶段 2 和阶段 3 不是必经之路——车主没有省钱意愿时可跳过
