# 价格与优惠（Which 条件）

帮车主了解价格、选择省钱方式，完成 Which 条件的收集。

## 场景判断

用户提到价格相关话题时，判断想了解什么：
- 行情价 → 行情参考价
- 门店报价 → 门店报价
- 省钱 / 便宜 / 优惠 → 省钱方式
- 只说"多少钱"且上下文无位置信息 → 一句话询问："您想先看看**市场行情参考价**，还是查一下**附近门店的实际报价**？"

## 行情参考价

只需要 What 就能查，用于了解大盘行情。

判定场景（无需询问）：
- 车主说"行情价"、"市场价"、"参考价"、"一般多少钱"
- 上下文中没有任何位置信息，也没有提到门店

执行：
```bash
cd <skill-dir> && python scripts/query_market_price.py --project-ids 502 505 --car-model-id ""
```

## 门店报价

需要 What + Where，用于比价和选店。

判定场景（无需询问）：
- 车主说"门店报价"、"店里多少钱"、"附近哪家便宜"
- 车主提到了具体位置或指定了门店
- 车主想比价、选店、下单

需要 shop_ids（门店 ID 列表），可从 `search_shops` 获取。

执行：
```bash
cd <skill-dir> && python scripts/query_shop_price.py --project-ids 502 505 --car-model-id "" --shop-ids S001 S002
```

## 省钱方式

根据项目类型判断适用的省钱方法：

### 平台九折

- 部分包含零部件更换的项目适用，需提前至少 2 天预订
- 通过供应链优化实现让利，不是补贴
- 优惠券部分可与九折叠加

### 保险竞价

- 保险类项目走 insurance-bidding skill，不在本 skill 处理
- 车主提到保险相关需求时，提示走保险竞价流程

### 商户自有优惠

- 部分商户有自己的折扣、满减、会员价等活动
- 根据项目类型、区域等条件查找有优惠的商户
- 可能与九折叠加

## 九折券金额计算

以预估总价的 1% 取整计算购券金额，抵扣金额为购券金额的 10 倍：

| 预估总价 | 购券金额 | 到店抵扣 | 起用门槛 |
|---------|---------|---------|---------|
| 800 元  | 8 元    | 80 元   | 800 元起 |
| 1250 元 | 12 元   | 120 元  | 1200 元起 |

到店结算时车主出示核销码抵扣。

## 用户不要优惠

车主明确拒绝省钱（如"不用了直接订"），Which 标记已满足，不再追问。
用户之后主动问价格时正常回答。

## 展示要求

- 直接用文字描述结果，不使用 spec 卡片
- 必须包含每个项目的具体价格数值（如"标准洗车（轿车）：30元"）
- 后续预订时需要用到这些价格

## 注意事项

- `<skill-dir>` 替换为 Skill 工具返回的实际路径，所有脚本必须在 skill 目录下执行
- `--car-model-id` 只能填上下文中已有的真实值，没有则填 ""（空字符串）
- "unknown"、"SUV"、"sedan" 等都属于编造，严禁使用
- 行情参考价和门店报价是两种独立查询，不要混淆
