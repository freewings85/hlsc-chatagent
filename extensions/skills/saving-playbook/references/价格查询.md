# 价格查询

帮车主了解项目价格，为后续省钱方案提供参考基准。

## 前置条件

- project_ids 已通过 `match_project` 获取
- car_model_id 必须来自上下文已有值，没有则传空字符串 ""，严禁编造

## 两种价格

### 行情参考价

市场平均价格，用于了解大盘行情。

判定场景（无需询问）：
- 车主说"行情价"、"市场价"、"参考价"、"一般多少钱"
- 上下文中没有任何位置信息，也没有提到门店

执行：
```bash
cd <skill-dir> && python scripts/query_market_price.py --project-ids 502 505 --car-model-id ""
```

### 门店报价

附近门店的实际报价，用于比价和选店。

判定场景（无需询问）：
- 车主说"门店报价"、"店里多少钱"、"附近哪家便宜"
- 车主提到了具体位置或指定了门店
- 车主想比价、选店、下单

需要 shop_ids（门店 ID 列表），可从 `search_shops` 或 `get_visited_shops` 获取。

执行：
```bash
cd <skill-dir> && python scripts/query_shop_price.py --project-ids 502 505 --car-model-id "" --shop-ids S001 S002
```

### 无法判断时

车主只说"多少钱"且上下文无位置信息时，一句话询问：
> "您想先看看这个项目的**市场行情参考价**，还是查一下**附近门店的实际报价**？"

不要过度解释两者区别，除非车主追问。

## 展示要求

- 直接用文字描述结果，不使用 spec 卡片
- 必须包含每个项目的具体价格数值（如"标准洗车（轿车）：30元"）
- 后续预订时需要用到这些价格

## 省钱引导

价格展示后，主动提一句：
> "价格大概是这样。要不要看看怎么更便宜？"

如果车主感兴趣，进入阶段 3（设计省钱方案）。
如果车主说"不用了"，直接进入阶段 4（选定商户）。

## 后续选项

价格展示后，告知车主可选的下一步：
1. **选商户报价预订** — 选择某个商户的报价直接预订（standard 模式）
2. **一口价委托预订** — 车主出一口价，委托推送给多家商户（commission 模式）
3. **多商户竞价预订** — 推送给多家商户竞价报价（bidding 模式）
4. **继续查报价** — 如查的是行情价，可继续查附近门店实际报价

## 注意事项

- `<skill-dir>` 替换为 Skill 工具返回的实际路径，所有脚本必须在 skill 目录下执行
- `--car-model-id` 只能填上下文中已有的真实值，没有则填 ""（空字符串）
- "unknown"、"SUV"、"sedan" 等都属于编造，严禁使用
- 行情参考价和门店报价是两种独立查询，不要混淆

## 工具

- `call_query_codingagent`：执行价格查询脚本

## 完成标准

- 已向车主返回对应类型的价格结果
- 已告知后续可选操作
