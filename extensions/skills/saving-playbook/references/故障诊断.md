# 故障诊断

车主描述故障症状时，通过故障现象库检索分析原因并推荐项目。

这是一个特殊入口，不是主流程。诊断完成后，诊断结果填入 What 条件继续推进。

## 判据

车主描述了车辆的异常现象或故障症状，需要分析原因。

典型话术：
- "刹车有异响"
- "方向盘抖动"
- "发动机故障灯亮了"
- "过减速带咚咚响"
- "车子开起来有点飘"
- "空调不制冷了"

## 流程

### Step 1：提取关键词

从车主描述中提取故障诊断关键词（故障症状、异常部件等）。

如果描述不够具体，引导车主补充：
- 什么时候出现的？
- 在什么情况下出现？（启动时、行驶中、刹车时等）
- 具体位置？（前面、后面、左边、右边）

### Step 2：检索诊断

获取关键词后，执行检索命令：

```bash
cd <skill-dir> && python scripts/search_project_by_keyword.py --keyword "刹车异响"
```

`<skill-dir>` 替换为 diagnose-car skill 的实际路径。

### Step 3：展示结果

将检索结果整理后展示给车主：
- 诊断标题和内容描述
- 可能的原因分析
- 关联的推荐项目

### Step 4：填入 What 条件

诊断结果中包含推荐项目时（通常会有 1-3 个关联项目）：
1. 向车主确认是否需要处理推荐的项目
2. 车主确认后，以推荐项目作为 What 条件的候选继续推进
3. 后续按正常流程收集其他 W 条件

## 多症状处理

车主可能同时描述多个症状：
- 分别提取每个症状的关键词
- 逐个检索，合并结果
- 如果多个症状指向同一个项目，合并说明
- 如果指向不同项目，按紧急程度排序推荐

## 注意事项

- 故障诊断是辅助手段，复杂故障仍需到店检查
- 不要给出过于确定的诊断结论，用"可能是"等表述
- 涉及安全部件（制动、转向、气囊等）时，必须强调到店检查
- 检索无结果时，建议车主到店做专业检查
- 不要用过于专业的术语吓到车主，用通俗语言解释
- 如果车主上传了图片或视频，结合视觉信息辅助判断
- 同一症状可能有多种原因，按可能性从高到低排列
