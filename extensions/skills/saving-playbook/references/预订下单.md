# 预订下单

4W 条件满足后，把已确认的项目、商户、时间、方案汇总成预订，完成下单。

## 两种预订模式

### 标准预订（standard）

车主选择某个商户的报价直接预订。

适用场景：What + Where(shop_id) + When 已满足。

流程：
1. 确认券型（10 元券 or 9 折券），两种都符合时默认 9 折券
2. 确认价格（车主已知目标价格，或给出行情价后车主同意）
3. 查看可用优惠：调 `search_coupon` 查看该商户是否有额外优惠（可与九折叠加），有则向车主展示
4. 确认服务时间（见下方"时间确认"）
5. 调用 `confirm_booking` 将预订信息发给前端确认（coupon_ids 传入用户选定的优惠券 ID，remark 传入车主备注）
6. 判断车主回复并处理（见下方"回复处理"）

### 委托预订（commission）

车主出一口价，委托推送给多家商户。

适用场景：What + Where(位置范围) + When + Which(一口价) 已满足。

流程：
1. 确认服务时间
2. 确认车主的一口价
3. 调用 `confirm_booking` 发给前端确认
4. 判断车主回复并处理

## agent 自动判断预订模式

| 信号 | 推断的模式 |
|-----|-----------|
| 用户选了商户 + 同意报价 + 确认时间 | standard |
| 用户给出目标价 / 要求代谈判 / 说"帮我谈" | commission |

## 过渡型方案

前置条件不全时的变通方案：

| 场景 | 处理方式 |
|-----|---------|
| 无 VIN | 先订简单项目（如洗车），让商户协助提供车架号 |
| 项目不确定 | 先订洗车 + 全面检查，到店后再确认 |
| 只要联系方式 | 给商户联系方式，车主自行线下对接（商户联系结束态） |
| 让商户主动联系 | 走 commission 模式，商户接单后会主动联系车主 |
| 想先谈价再订 | 话痨代谈判，达到目标后再标准预订 |
| 车主已在现场 | 确认商户是否认证，是则直接走标准流程 |

## 工具

- `confirm_booking`：汇总预订信息发给前端，返回车主回复文本。参数包括 project_ids、shop_ids、car_model_id、booking_time、price、coupon_ids（优惠券 ID 列表，来自 search_coupon）、remark（车主备注）
- `search_coupon`：查看商户可用优惠，返回的 coupon_id 传入 confirm_booking 的 coupon_ids

## 完成标准

- 预订信息已确认提交（或车主取消）
- 结果已告知车主
