# 商户复杂查询

当 search_shops 无法直接满足用户需求时，用 `call_query_codingagent` 完成复杂查询。

**只提取用户说了的条件，未提及的不写、不编造。**

## 可用维度

| 维度 | 说明 |
|------|------|
| 城市 | 城市名称 |
| 区域/地址 | 区、街道、路名、地标等 |
| 距离 | 具体距离数字，或按距离排序 |
| 商户名称 | 指定的店名或品牌 |
| 商户类型 | 业态类型 |
| 项目/服务 | 用户要做的养车项目 |
| 评分 | 评分门槛，或按评分排序 |
| 营业时间 | 要求的营业时段 |
| 价格/报价 | 价格上限，或按价格排序、比价 |
| 优惠 | 是否有优惠，或按优惠力度排序、计算优惠后价格 |
| 人气/成交量 | 按成交量排序 |

## 调用 call_query_codingagent

用结构化 Markdown 编写 query，从对话中提取用户实际提到的条件填入：

```md
# 查询目标
（用户要什么）

# 已知条件
- （用户提到的条件）

# 必须满足
- （硬性条件）

# 返回结果
- （排序方式 + 每条包含哪些信息）
```
