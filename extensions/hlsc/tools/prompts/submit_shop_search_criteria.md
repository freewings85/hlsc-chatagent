把用户当前表达的"找店条件"静默登记到会话，由系统去查商户并把结果带回来。

这是一次**内部登记**——调用对用户不可见。不要在回复里预告"我帮你搜一下""保存中"，
也不要把 `shop_search_info` 的内容作为 JSON 或清单贴给用户看。用户看到的只是你对他说的话，
搜索是你和系统之间的事。

## 何时调

- 用户给的找店条件已经清晰到能组出一个可查的 `query`（参数 schema 里的规则）
- 用户补充 / 修正之前给过的条件 → 用新值覆盖，整包 `shop_search_info` 重新提交

## 何时不调

- 用户只是闲聊 / 提问 / 表达情绪，没给出新找店条件
- 条件模糊到 `query` 还不成立（比如只说"帮我找个店"）→ 先追问，不要塞空壳
- 用户明确是别的意图（查优惠、问价格等）→ 跟用户走，不要硬凑搜索

## 返回值

登记完会回一段文字，比如"找到 3 家商户"或"未找到匹配的商户"，是给你判断下一步用的
线索——不是给用户看的原话，按对话语气转述。真实的商户列表会通过前端事件直接渲染出来，
你不用把它复述给用户。

## 参数

`shop_search_info` 一个字段，形状见它的描述。按当前对话里已确认的信息来填，
用户没讲的字段就留空——不要编。
