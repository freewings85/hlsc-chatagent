说明：
这个工具只用于商户复杂查询兜底。先尝试普通商户查询；只有普通查询无法满足时，才把复杂的商户筛选、比较或结果整理任务交给 QueryCodingAgent。

When to use:
- 已经明确是在查商户，并且普通商户查询无法直接完成当前任务。
- 需要多条件组合筛选、排序、聚合、比较，或需要对商户结果做进一步计算整理。
- 主流程需要先拿到复杂商户查询结果，再继续做解释、推荐或确认。
- 不要用于普通商户查询，例如：普通附近搜索、单一条件筛选、按 `shop_id` 查详情。
- 不要用于非商户查询。

Usage notes:
- 优先使用普通商户查询能力；只有普通查询结果不够，或查询条件本身已经超出普通查询能力时，再调用这个工具。
- 传给它的 `query` 要简洁、完整、可执行，建议用结构化 Markdown 描述，不要写成散文。
- 至少说明：查询目标、已知条件、必须满足的条件、优先考虑的条件、希望返回的结果。
- 已知的数值条件必须直接展开写进去，不要写成“我当前位置”“附近”“默认车型”这类相对表达。
- 例如：有经纬度就写 `latitude` / `longitude`，有距离就写 `radius_km` 或 `distance_km`，有 `project_id` / `car_model_id` / `shop_id` 就直接写明。
- 它只负责商户查询和结果整理，不负责业务推荐、用户沟通或最终决策。
- 不要把含糊的问题直接交给它；主流程应先把条件补清楚，再调用。
- 如果主流程还缺关键数值条件，就先补清楚，不要让它自己猜。

Example usage:
```md
# 查询目标
筛出一组适合洗车的商户，并做结果整理

# 已知条件
- latitude: 31.1794
- longitude: 121.4493
- radius_km: 10

# 必须满足
- 支持洗车

# 优先条件
- 评分 4.5 以上
- 距离更近优先
- 返回时按距离和评分综合排序

# 返回结果
- 返回 5 家商户
- 每家包含名称、距离、评分
- 说明入选原因
```
