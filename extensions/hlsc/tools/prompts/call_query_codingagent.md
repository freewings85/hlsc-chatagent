说明：
这个工具只用于商户复杂查询兜底。先尝试普通商户查询；只有普通查询无法满足时，才把复杂的商户筛选、比较或结果整理任务交给 QueryCodingAgent。

When to use:
- 已经明确是在查商户，并且普通商户查询无法直接完成当前任务。
- 需要多条件组合筛选、排序、聚合、比较，或需要对商户结果做进一步计算整理。
- 不要用于普通商户查询，例如：普通附近搜索、单一条件筛选。

Query 编写规则：
- 用结构化 Markdown 描述，至少说明：查询目标、已知条件、必须满足的条件、返回结果。
- 已知的数值条件必须直接写明，不要用"我当前位置""附近"这类相对表达。
- 只写用户提供的或系统已知的条件，不要自己编造数值（如用户没说距离就不要写距离，由 coding agent 用 API 默认值）。
- 缺少的条件留空或不写，由 coding agent 使用默认值处理。
- 返回结果最多 10 条，有多少返回多少，不要因为数量不足就拒绝返回。

Example:
```md
# 查询目标
找出***附近***最便宜的商户

# 已知条件
- 地址：***

# 必须满足
- 支持***项目

# 返回结果
- 返回找到的商户（按价格从低到高排序）
- 每家包含名称、距离、评分、***价格
```
