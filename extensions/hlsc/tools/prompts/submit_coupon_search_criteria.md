把用户当前表达的"找优惠活动条件"静默登记到会话，由系统去查优惠活动并把结果带回来。

这是一次**内部登记**——调用对用户不可见。不要在回复里预告"我帮你搜一下""保存中"，
调用过程和参数内容都不展示给用户。回复用自然中文的业务语言。

## 何时调

- 用户给的找优惠活动条件已经清晰到能组出一个可查的查询
- 用户补充 / 修正之前的条件时，把所有已确认的条件合并后**整包重新提交**
  （完整覆盖，不是增量 patch）

字段不分"归活动还是归商户"——按字段语义落进对应参数即可，后端会自己按语义
拆成活动维度和商户维度。

## 何时不调

- 用户只闲聊 / 表达情绪 / 问非优惠活动相关问题
- 用户明确是别的意图（预订下单、查商户评分等）

## 返回值

登记完会回一段文字（如"已为用户展示 3 条优惠活动"或"未找到匹配的优惠活动"）——
给你判断下一步用的线索，不是给用户看的原话。真实活动详情通过前端事件直接渲染，
你**不要复述活动文案**，按对话语气说一句自然的回应即可。
