把用户给出的业务信息提交给后台工作流。工作流收到后可能会处理并返回结果。

## 何时调

- 从用户消息里拿到清晰、已确认的事实（姓名、地址、车架号、选中项等）
- 用户纠正或补充之前给过的信息
- 用户在一轮里给了多个事实 → 打包一次性传入
- 用户明确表示拒绝或跳过某个字段 → 用一个明确的标记传出去

## 何时不调

- 用户只在提问 / 闲聊 / 表达情绪，没给出新事实
- 用户要你执行一个业务动作（查询、领取、预订等）→ 那是做这件事的工具的活
- 字段值模糊或你还在猜测 → 先和用户确认清楚，不要塞不确定的值
- 当前这一步的提示和用户想做的事不一致 → 优先跟用户走，不要为了"回应提示"塞脏字段

## 返回值

工作流处理完后会返回一段文字。这段文字可能是简单的确认，也可能包含
工作流处理后的业务结果（如搜索结果摘要、状态变更说明）。根据返回内容决定下一步
怎么回复用户。

## 参数

- 传一个字段字典，值可以是字符串、数字、布尔，也可以是嵌套对象或数组（比如复杂查询结构、列表选项）。按当前步骤要求的字段形态来
- 清除某个字段：传 null
- 一次清除多个条件重来：一次把相关字段都传 null
