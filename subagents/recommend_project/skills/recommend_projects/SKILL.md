# 养车推荐项目 Skill

## 执行流程

### 1. 收集车辆信息并查询推荐项目

从用户描述和 request_context 中提取 VehicleInfo：

- **car_model_id**：上下文已有 → 直接用；用户提到车型 → 调 fuzzy_match_car_info；都没有 → 可选
- **car_age_year**：用户提到 → 直接换算成年份；没有 → 可选
- **mileage_km**：用户提到 → 直接使用；没有 → 可选

直接调 `recommend_projects` 查询。car_model_id 为空时 API 自动使用样本 VIN 推荐，不阻塞流程。

### 2. 展示项目列表并引导选择

调 `ask_user_select_project` 将项目列表和 vehicle_info 展示给用户，等待用户回复。

### 3. 判断用户回复

根据用户回复的文字内容判断意图：

- **选择了某个项目**（用户提到了列表中某个项目的名称）→ 确认选择，skill 完成
- **想退出**（"不用了"、"算了"、"取消"、"不需要"等）→ 告知用户，skill 完成
- **提到了车龄或里程**（如"我的车3年了"、"跑了5万公里"）→ 提取信息更新 vehicle_info，引导用户选择项目，回到步骤 3
- **无关内容** → 简要引导用户从列表中选择，回到步骤 3

## 完成标准

- 用户选中了一个项目 → 返回选中项信息
- 用户明确表示不需要 → 结束
