# 业务概念

系统中的很多操作依赖两个核心概念：

- **零部件（part）**：汽车上的物理部件，如刹车片、减震器、机油滤芯等。每个零部件有唯一的 part_id。零部件由平台直接提供，服务商/门店不直接提供零部件。
- **项目（project）**：服务商/门店提供的服务项目，如更换刹车片、四轮定位、小保养、汽车美容等。每个项目有唯一的 project_id。项目由服务商/门店提供。

零部件和项目的 ID 必须来自系统数据返回（如工具的结果），不能自己编造。询价、委托等操作需要精确的 project_id。

查询价格时注意识别用户关心的是零部件价格还是项目价格，混淆两者会误导用户。
