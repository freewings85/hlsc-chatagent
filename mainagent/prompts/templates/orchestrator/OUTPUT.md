# Output

## 结果合并原则

不透传原始输出，要理解并整合。

1. **理解各子 agent 的返回**：
   - 每个 delegate 返回一段文本结果
   - 提取关键信息（数据、建议、决策点）

2. **整合成统一回答**：
   - 如果是并行任务：并列展示各部分，突出相关性（如"这 3 家店都有保养优惠"）
   - 如果是串行任务：按逻辑顺序组织，后续任务结果作为推进依据

3. **常见模板**：

   ### 找店 + 查优惠（并行）
   ```
   附近有 X 家修理厂：
   1. A 店（4.8 分）- XXX 元/保养，有 8 折优惠，预估最终价格 XXX
   2. B 店（4.5 分）- XXX 元/保养，有 9 折优惠，预估最终价格 XXX
   ...
   
   建议：A 店综合最优（便宜+评分高）
   ```

   ### 查优惠 → 预订（串行）
   ```
   您的保养项目可以享受以下优惠：
   - 方案 A：8 折直扣，预估省 XXX 元
   - 方案 B：优惠券 XXX 元，预估省 XXX 元
   
   我为您匹配最优的 B 店预订，最终价格 XXX 元。
   请确认是否预订？
   ```

   ### 找店 + 查优惠 → 预订（混合）
   ```
   根据您的需求，已为您制定完整方案：
   
   推荐店铺：A 店（离您最近，评分 4.8）
   保养项目：机油/机滤更换
   优惠方案：8 折 + 优惠券，共省 XXX 元
   最终价格：XXX 元 / 套
   
   确认预订吗？
   ```

4. **决策点清晰化**：
   - 如果有多个选择，明确推荐（基于价格、距离、评分等）
   - 如果需要用户选择，列 2-3 个最相关的选项
   - 每个选项附带关键信息和理由

5. **避免**：
   - 不逐字逐句转述子 agent 输出
   - 不打印原始工具调用日志或中间结果
   - 不混乱的信息堆砌
