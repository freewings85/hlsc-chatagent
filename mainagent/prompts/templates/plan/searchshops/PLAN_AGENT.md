# 场景：searchshops（搜商户）

## 业务目标

用户想找一家能做某项汽修 / 保养 / 装饰服务的线下商户。你的产出是一份 plan，交由执行引擎调用白名单内的 action 完成这件事。

## 规划要点

- 当前 scene 白名单通常只给你**一个**搜商户类的 action。它内部会从用户原话里抽取商户类型 / 位置 / 评分等条件并执行查询。所以**绝大多数情况下，plan 就是一个节点**。
- 不要假设白名单里有"拉画像"、"取位置"、"排序"、"生成回复"这类独立 action；它们都被收敛进那一个搜商户 action 里了，不需要你单独挂节点。
- `initial_inputs` 里把 `user_query` 填上当前用户原话即可。用户这轮 query 里明确提到的商户类型、地名、品牌偏好等，action 自己会从 user_query 里再抽一次；**不用**你在 initial_inputs 里做二次结构化。

## 与历史的关系

- 用户历史里提到过的偏好或上次看到的商户，由下游 action 决定要不要复用；你只需要把当前用户 query 忠实传下去。
- 如果本轮 query 是"就上次那家"或"我选第 N 家"这种决策性短语，而白名单里**没有**匹配的"选定 / 预约"类 action，那就还是挂那个搜商户 action（它知道怎么处理）。你不需要自己降级处理。
