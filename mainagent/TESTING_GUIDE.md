# SearchCoupons 场景测试指南

本文档为测试人员（sc-val-tester）提供完整的 searchcoupons 场景测试清单和步骤。

## 前置准备

### 步骤 1：启动 mainagent 服务

```bash
cd mainagent

# 方式 A：使用脚本启动（推荐）
./start_local.sh

# 方式 B：直接启动
uv run python server.py

# 方式 C：指定端口
uv run python server.py --port 8100
```

**验证服务启动**：
```bash
curl http://localhost:8100/health
# 返回 200 OK 表示服务已启动
```

### 步骤 2：验证 Mock 数据

```bash
cd mainagent
uv run python verify_mock_data.py
```

**预期输出**：
```
✓ 全部通过 (3/3)
🎉 Mock 数据已就绪，可以启动 mainagent 进行测试
```

## 测试用例清单

### 场景 1：查优惠 - 有商户优惠 + 平台优惠（对比）

**前置条件**：
- mainagent 已启动（8100）
- 用户有明确的养车项目

**测试步骤**：

| 轮次 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 1 | "我要保养，想找优惠，要支付宝的，送洗车最好" | 调用 `search_coupon`，组装 semantic_query | ✓ semantic_query 包含"支付宝、送洗车" |
| 2 | （观察结果） | 返回 CouponCard（至少 3 条商户优惠） | ✓ 显示具体金额和条件 |
| 3 | "这个机油 8 折不错，帮我申请" | 确认到店时间（如果用户未说明） | ✓ 询问"什么时候可以去？" |
| 4 | "周三下午 2 点" | 调用 `apply_coupon`，生成联系单 | ✓ 返回订单号、商户、活动名 |

**验证清单**：
- [ ] semantic_query 正确提取所有偏好（支付方式、赠品等）
- [ ] CouponCard 显示正确的活动名称、优惠金额、使用条件
- [ ] apply_coupon 前确认了到店时间
- [ ] 返回的联系单包含订单号、商户、时间等必要信息

### 场景 2：查优惠 - 仅平台优惠（无商户优惠降级）

**前置条件**：
- mainagent 已启动
- 使用一个没有商户优惠的场景

**测试步骤**：

| 轮次 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 1 | "查一下有什么优惠活动" | search_coupon 返回空商户优惠 | ✓ shopActivities = [] |
| 2 | （观察） | Agent 主动介绍平台 9 折方案 | ✓ "没有直接商户优惠，但可以通过话痨预订享 9 折" |
| 3 | 用户表示感兴趣 | 引导进入预订流程或继续找优惠 | ✓ 自然转场，不生硬 |

**验证清单**：
- [ ] 检测到无商户优惠时自动降级
- [ ] 介绍九折方案的文案自然流畅
- [ ] 不刻板地说"虽然没有 XXX 工具"这样的话

### 场景 3：查优惠 - 完全无优惠

**前置条件**：
- 通过 mock 服务器 + semantic_query="empty" 或其他无结果场景

**测试步骤**：

| 轮次 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 1 | "我要找轮胎优惠" | search_coupon 返回空 | ✓ platformActivities=[], shopActivities=[] |
| 2 | （观察） | Agent 读取 saving-methods，介绍四种省钱方式 | ✓ 介绍找新商户、保险竞价等方向 |
| 3 | 用户选择一种方式 | 自然引导到该方式的下一步 | ✓ 流程清晰 |

**验证清单**：
- [ ] 无优惠时读取 saving-methods skill
- [ ] 介绍的四种方式是否相关和完整
- [ ] 转向其他省钱方式时是否自然

### 场景 4：申领优惠 - 确认到店时间

**前置条件**：
- Agent 已展示优惠卡片

**测试步骤**：

| 情况 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 4a | "帮我申请这个，我周三下午可以" | 直接调用 apply_coupon（已有时间） | ✓ 不重复问时间 |
| 4b | "帮我申请" | Agent 先问"什么时候方便？" | ✓ 确认时间后再调用 |
| 4c | 用户没给具体时间 | Agent 二次确认（"下午 2 点还是 3 点？"） | ✓ 获得具体时刻 |

**验证清单**：
- [ ] 若用户已说时间，无需重复确认
- [ ] 若用户未说时间，及时询问
- [ ] 时间格式转换是否合理（自然语言 → HH:MM）

---

### 场景 5：转预订（从申领到完整预订）

**前置条件**：
- 用户已申领优惠或表示想预订

**测试步骤**：

| 触发条件 | 用户输入 | Agent 预期行为 | 验证点 |
|--------|--------|--------------|------|
| 5a | 申领后问接下来 | "已帮你预约，接下来怎么样？" | Agent 说"我帮你安排完整预订" |
| 5b | 主动说想预订 | "这个优惠不错，直接帮我订吧" | Agent 识别预订意图，自然转场 |
| 5c | 对比后选最划算 | "我看了这几个，直接给我订最划算的" | Agent 确认选择，进入预订流程 |

**验证清单**：
- [ ] 转场时是否说明"进入预订流程"
- [ ] 转场是否自然流畅（不卡顿、不生硬）
- [ ] 下一轮是否真的进入预订相关问题

---

### 场景 6：按金额排序（优惠力度对比）

**前置条件**：
- search_coupon 返回多条优惠，按金额排序

**测试步骤**：

| 轮次 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 1 | "找优惠，我最关心能省多少钱" | semantic_query 中体现"高优惠优先" | ✓ 调用时 sort_by="discount_amount" |
| 2 | （观察结果） | 优惠金额从高到低展示 | ✓ 第一个最划算的优先展示 |

**验证清单**：
- [ ] semantic_query 中是否正确表达"金额优先"意图
- [ ] 返回结果是否按优惠金额降序排列
- [ ] CouponCard 中是否清晰展示每个优惠的具体金额

---

### 场景 7：即将过期优先（时间敏感）

**前置条件**：
- 有多条优惠，部分即将过期

**测试步骤**：

| 轮次 | 用户输入 | Agent 预期行为 | 验证点 |
|------|--------|--------------|------|
| 1 | "有什么快要过期的优惠吗？" | semantic_query 体现"时间优先" | ✓ sort_by="validity_end" |
| 2 | （观察结果） | 优先显示即将过期的活动，并强调期限 | ✓ "仅剩 X 天"提示 |

**验证清单**：
- [ ] semantic_query 是否正确识别时间敏感意图
- [ ] 返回结果是否按过期时间升序（最紧急的优先）
- [ ] 是否特别提示"仅剩 X 天"等紧急信息

---

### 场景 8：错误处理 & 边界情况

**测试步骤**：

| 编号 | 场景 | 用户输入 | Agent 预期行为 | 验证点 |
|-----|------|--------|--------------|------|
| 8a | 工具失败 | （模拟 DataManager 超时） | 返回友好错误提示，建议重试或其他方案 | ✓ 不暴露技术错误 |
| 8b | 用户未说时间 | "帮我申请" | Agent 主动问 | ✓ 必须确认时间后才调用 apply_coupon |
| 8c | 无效 ID | 用户尝试申请不存在的优惠 | 拒绝或重新查询 | ✓ 不编造数据 |
| 8d | 用户拒绝优惠 | "这些都不符合我的预期" | 不强推，尊重用户选择 | ✓ 转向其他方案 |

**验证清单**：
- [ ] 出错时消息清晰，无技术细节泄露
- [ ] 工具参数校验（activity_id、shop_id 必须来自真实查询结果）
- [ ] 用户拒绝时是否转向降级方案（saving-methods）

## 遇到问题的排查步骤

```bash

curl http://localhost:8100/health

# 检查 2：是否是 mock 模式
grep MOCK_SEARCH_COUPON mainagent/.env.local

# 检查 3：查看日志
tail -f mainagent/logs/*
```
